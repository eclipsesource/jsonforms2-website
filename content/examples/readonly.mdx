## Form Wide

There are multiple ways to use the `readonly` option.
One is to define `readonly` on the JsonForms object itself. This will disable *all* inputs of this form.

```js
<JsonForms
  renderers={materialRenderers}
  cells={materialCells}
  data={data}
  schema={schema}
  uischema={uischema}pp
  readonly
/>
```

Although the example above shows a react example, this flag also exists for angular and vue.

## Input based

### UI Schema

If you want to set the `readonly` option on a per-input basis, you can do so by setting `readonly: true` on the element options within the UI schema:

```js
{
  type: "VerticalLayout",
  elements: [
    {
      type: "Control",
      label: "Completed",
      scope: "#/properties/firstname",
      options: {
        readonly: true
      }
    }
  ]
}
```

### Schema

An alternative to set the `readOnly` option via the UI Schema, is to set this option via the schema:

```js
{
  properties: {
    firstname: {
      type: "string",
      readOnly: true
    },
  }
}
```

Note: `readonly` will be ignore within the schema, only `readOnly` is a [valid option](https://json-schema.org/draft/2020-12/json-schema-validation.html#rfc.section.9.4).

### Rule

The last option to disable an input is to set a `DISABLE` (or `ENABLED` with a false condition) rule on it.
Read more about [rules](/docs/uischema/rules).

```js
{
  type: "VerticalLayout",
  elements: [
    {
      type: "Control",
      label: "Completed",
      scope: "#/properties/firstname",
      rule: {
        effect: "DISABLE",
        condition: {
          scope: "#/properties/name",
          schema: {} //always true
        }
      }
    },
    // OR
    {
      type: "Control",
      label: "Completed",
      scope: "#/properties/firstname",
      rule: {
        effect: "ENABLE",
        condition: {
          scope: "#/properties/name",
          schema: { not: {} } //always false
        }
      }
    }
  ]
}
```
